# 도란도란 기능명세서

> 단절을 겪는 두 세대를 잇는 정기 대화 매칭 서비스
은둔 청년의 사회참여 활동 모델 + 어르신 말동무 서비스
> 

---

# 1. 사용자 구분

| 구분 | 설명 |
| --- | --- |
| 관리자 | 서비스 운영, 청년 가입 검수(승인/반려), 악성 유저 제재(Ban), 매칭 관리, 신고 처리, 증명서 발급, 기기 상태를 관리하는 사용자 |
| 청년 | 관리자 승인 후 등록된 어르신 정보를 조회하고, 담당 가능 인원 내에서 말동무 활동 대상을 선택하여 정기 대화를 제공하는 사용자 |
| 어르신 | 보호자가 등록한 정보를 바탕으로 무상 제공된 전용 기기를 통해 청년과 화상/음성 대화를 이용하는 사용자 |
| 보호자/기관 | 어르신 정보 등록, 기기 배송 관리, 기본 진행 여부 확인을 담당하는 대리인 |

---

# 2. 공통 기능

## F-01 사용자 유형 선택

- **설명**: 사용자가 서비스 이용 목적에 따라 청년 또는 보호자 유형을 선택하는 기능
- **사용자 시나리오**: 첫 화면에서 '청년으로 시작하기' 또는 '보호자로 시작하기'를 선택하여 역할별 가입 폼으로 이동한다.
- **입력**: 사용자 유형 선택값
- **출력**: 역할별 회원가입 화면 분기
- **비고**: 회원가입 완료 후 청년은 청년 자기소개 프로필 등록(F-06)으로, 보호자는 어르신 프로필 등록(F-11)으로 분기 이동한다.

## F-02 회원가입

- **설명**: 청년 또는 보호자가 필수 정보를 입력하여 서비스 계정을 생성하는 기능
- **처리**: 필수값 검증 후 계정 생성. (※ 청년 계정은 가입 직후 기본 상태가 `PENDING(승인 대기)`으로 설정됨)
- **예외**: 중복 이메일, 필수값 누락 시 오류 메시지 출력
- **비고**: 휴대폰 번호를 필수 입력받아 별도 인증 절차 없이 본인 확인을 갈음한다.

## F-03 로그인 (가입 승인 및 제재 상태 검증)

- **설명**: 사용자가 이메일과 비밀번호를 입력하여 서비스에 접속하는 기능
- **처리**: 계정 인증 후 사용자 역할 및 계정 상태(`status`), 가입 승인 상태(`approval_status`) 확인
- **예외 (핵심)**:
    - `approval_status == 'PENDING'`: 메인 홈 진입을 차단하고 "관리자 승인 대기 중입니다" 팝업 출력
    - `status == 'SUSPENDED'`: "운영 정책 위반으로 이용이 제한되었습니다" 제재 안내 출력
- **MVP 처리**: 데모에서는 시연 흐름을 위해 더미 계정으로 로그인 과정을 간소화(프리패스)하여 보여준다.

## F-04 SNS 간편 로그인

- **설명**: 카카오, 네이버, 구글 계정을 이용하여 간편하게 로그인/가입하여 은둔 청년의 가입 허들을 낮추는 기능
- **MVP 처리**: 프론트엔드로만 구현한다.

## F-05 비밀번호 찾기

- **설명**: 등록된 이메일 또는 휴대폰 번호를 통해 임시 비밀번호 또는 재설정 링크를 발송하는 기능

# 3. 청년용 자기소개 기능

> 청년 회원가입(F-02) 완료 직후 진입하는 분기 흐름이다.
> 

## F-06 자기소개 프로필 등록 및 승인 대기

- **설명**: 청년이 말동무 활동을 위해 자신의 기본 소개 정보를 등록하고 관리자 승인을 대기하는 기능
- **사용자 시나리오**: 청년이 프로필 사진, 키워드, 인사말을 입력하고 저장하면, 프로필이 승인 대기(`PENDING`) 상태로 관리자에게 전송된다.
- **처리**: 입력값 검증 후 `youth_profiles` 테이블에 정보 저장 및 상태 업데이트
- **예외**: 관리자가 반려(`REJECTED`)한 경우 반려 사유 노출 및 재수정 권고

## F-07 프로필 사진 등록

- **설명**: 본인이 편안함을 느끼는 이미지나 기본 제공 일러스트/캐릭터를 프로필 사진으로 업로드

## F-08 나의 키워드 선택

- **설명**: `#차분한`, `#이야기_잘듣는` 등 자신의 성향과 관심사를 태그 형태로 선택하여 매칭 참고 자료로 활용 (경쟁 지표 아님)

## F-09 한 줄 인사말 작성

- **설명**: 어르신께 전하고 싶은 짧은 첫인상 문구 작성 (비속어 필터링 적용)

## F-10 목소리 샘플 등록 (선택)

- **설명**: 짧은 음성을 녹음하여 매칭 전 어르신(보호자)이 목소리를 확인할 수 있도록 제공

# 4. 어르신 프로필 등록 기능

> 보호자 회원가입(F-02) 완료 직후 진입하는 분기 흐름이다. 청년 자기소개(F-06)와 짝을 이루는 가입 분기의 한 축이다.
> 

## F-11 어르신 프로필 무상 등록

- **설명**: 보호자(또는 복지기관)가 회원가입 직후, 말동무 서비스를 이용할 어르신의 프로필과 대화 조건을 등록하는 기능. 청년 자기소개 페이지와 동일한 구조를 사용하되 어르신 전용 항목을 추가한다.
- **사용자 시나리오**: 보호자가 회원가입을 마친 뒤 어르신 프로필 등록 페이지로 이동한다. 어르신의 사진, 키워드, 인사말과 대화 조건을 입력하고 저장하면 어르신이 매칭 가능 상태로 등록된다.
- **입력**:
    - (공통 항목 — 청년 프로필과 동일) 프로필 사진, 키워드, 한 줄 인사말
    - (어르신 전용 항목) 이름, 연령대, 성별, 주소, 대화 가능 시간, 활동 난이도, 보호자 요청 사항
- **처리**: 필수값 검증 후 `seniors` 테이블에 정보 저장 및 매칭 가능 상태(`AVAILABLE`)로 등록
- **출력**: 어르신 프로필 등록 완료 화면 및 등록된 프로필 카드 미리보기
- **예외**: 필수값 누락, 대화 가능 시간 미입력 시 저장 제한
- **비고**:
    - 어르신 프로필은 청년 프로필과 동일한 구조를 사용하되 목소리 샘플 항목은 제외한다.
    - 청년에게는 주소, 연락처 등 민감한 개인정보를 블라인드 처리하여 매칭에 필요한 범위만 노출한다.
    - 어르신은 별도 계정을 생성하지 않으며 보호자 계정에 종속된다.

# 5. 무상 서비스 관리

> 기존 유료 결제/구독 관련 기능(구독 플랜 선택, 결제 수단 등록, 정기 구독 관리)은 전면 삭제되었다. 본 서비스는 무상 제공을 원칙으로 한다.
> 

## F-12 전용 기기 배송/등록 조회

- **설명**: 어르신에게 무상으로 제공되는 전용 태블릿의 배송 상태(`READY`/`SHIPPING`/`DELIVERED`) 및 연동 상태를 확인하는 기능
- **사용자**: 보호자, 관리자
- **출력**: 배송 상태, 기기 연동 상태
- **MVP 처리**: 실제 배송사 연동은 제외하고 시연용 상태값으로 표시한다.

# 6. 청년 선택형 매칭 기능

## F-13 어르신 목록 조회

- **설명**: 청년이 등록된 어르신 목록을 대화 가능 시간, 관심사, 대화 방식 등으로 필터링하여 조회하는 기능

## F-14 어르신 상세 정보 보기

- **설명**: 청년이 특정 어르신의 상세 대화 조건과 보호자 요청 사항을 확인하는 기능

## F-15 청년 선택형 매칭 생성 및 사전 인사 (수락 대기 없음)

- **설명**: 청년이 담당 가능 인원 내에서 어르신을 선택하고 첫인사말을 작성하여 즉시 매칭을 성립시키는 기능
- **사용자 시나리오**: 청년이 어르신을 선택한 뒤 '사전 인사말(Ice-breaking)' 모달창에 안부 인사를 작성하고 완료를 누른다.
- **처리**: 별도의 어르신/보호자 수락 절차 없이 즉시 상태값을 `MATCHED`로 생성하고 사전 인사말을 DB에 저장한다.
- **예외**: 담당 가능 인원 초과 시 매칭 제한 알럿 출력

## F-16 담당 인원 제한 관리

- **설명**: 청년 1인당 담당 가능한 최대 어르신 수(예: 최대 5명, 그 아래로는 자유롭게 매칭)를 동일하게 제한하여 과도한 경쟁을 방지하는 기능

## F-17 매칭 상태 관리

- **설명**: 매칭 완료(`MATCHED`), 진행 중(`IN_PROGRESS`), 종료(`ENDED`) 등 매칭 생애주기를 관리

# 7. 일정 관리 기능

## F-18 가능 시간 등록

- **설명**: 청년은 활동 가능 시간을, 보호자는 어르신의 대화 가능 시간을 캘린더에 등록하는 기능

## F-19 일정 확정

- **설명**: 매칭된 양측의 가능 시간이 겹치는 슬롯 중 하나를 확정하여 정기 대화 일정을 생성하는 기능

## F-20 일정 조회

- **설명**: 사용자가 예정된 대화 일자, 시간, 상대방 정보를 확인하는 기능

## F-21 리마인드 알림

- **설명**: 대화 시작 10분 전 양측 기기로 Push 알림 또는 인앱 알림 전송

# 8. 어르신 전용 기기 기능 (단순화 UI)

## F-22 전용 기기 메인 화면

- **설명**: 제공된 전용 태블릿을 켜면 복잡한 조작 없이 직관적인 큰 버튼 위주의 전용 UI가 표시되는 기능

## F-23 얼굴 보며 전화하기 (화상)

- **설명**: 커다란 '페이스톡' 버튼 터치 한 번으로 매칭된 청년과 화상 통화 연결

## F-24 목소리만 듣기 (음성)

- **설명**: 커다란 전화기 모양 버튼 터치 한 번으로 음성 통화 연결

## F-25 도움 요청하기

- **설명**: 긴급 상황 발생 시 관리자/보호자에게 호출 신호 및 기기 상태 전송

## F-26 화면 스와이프 메뉴 전환

- **설명**: 터치가 어려운 어르신을 위해 화면을 좌우로 크게 밀어 메뉴를 전환하는 기능

## F-27 일정 기반 통화 참여 유도

- **설명**: 대화 시간이 임박하면 "곧 대화가 시작됩니다"라는 큰 팝업과 통화 참여 버튼을 자동 표시

# 9. 활동 기록 및 증명서 기능

> 기존 유료 현금성 정산 관리 기능은 삭제되었다. 현금 보상 대신 사회참여 활동 증명서를 제공한다.
> 

## F-28 활동 기록 작성 (봉사 시간 누적 트리거)

- **설명**: 청년이 대화 종료 후 대화 진행 여부, 실제 대화 시간(분), 특이사항을 기록하는 기능
- **처리**: 기록 제출 즉시 백엔드에서 해당 청년의 `total_duration_minutes(누적 활동 시간)` 데이터에 실제 대화 시간을 플러스(+) 합산한다.
- **비고**: 보호자의 평가 지표가 아니라, 공인 시간 인정 및 진행 여부 확인용으로만 사용된다.

## F-29 활동 기록 확인

- **설명**: 보호자가 어르신의 대화가 잘 진행되었는지 기본 진행 여부와 단순 특이사항만 조회하는 기능 (청년 평가/평점 시스템 없음)

## F-30 사회참여 활동 증명서 발급

- **설명**: 청년의 자존감 회복 및 이력 증빙을 위해 누적된 활동 시간을 공식 증명서로 발급하는 기능
- **사용자 시나리오**: 마이페이지에서 총 누적 시간을 확인하고, 기준 시간(예: 10시간) 도달 시 증명서 발급 버튼을 누른다.
- **처리**: 시스템이 고유 발급 시리얼 번호(`DRDR-2026-XXXX`)를 생성하고 공인 수료증 PDF 다운로드 링크를 제공한다.

# 10. 안전장치 및 관리자 기능

## F-31 신고하기

- **설명**: 부적절한 대화나 비상 상황 발생 시 청년/보호자/어르신이 관리자에게 신고를 접수하는 기능

## F-32 매칭 중단 요청

- **설명**: 성향 불일치 등으로 매칭 유지가 어려울 경우 사유를 작성해 관리자에게 매칭 종료를 요청하는 기능

## F-33 관리자 가입 검수 및 악성 유저 제재(Ban)

- **설명**: 은둔 청년의 가입을 통제하고, 신고 접수된 악성 유저의 접근을 영구 차단하는 관리자 최고 권한 기능
- **사용자 시나리오 (승인)**: 관리자가 대시보드에서 `PENDING` 상태의 청년 프로필을 검토 후 `APPROVED`로 변경하거나 반려(`REJECTED`) 사유를 적어 돌려보낸다.
- **사용자 시나리오 (제재)**: 모니터링을 통해 적발된 유저의 상태값을 `SUSPENDED`로 강제 변경하여 즉시 모든 로그인을 차단한다.

## F-34 신고 및 중단 요청 처리

- **설명**: 관리자가 접수된 신고(F-31) 및 매칭 중단 요청(F-32) 사실관계를 파악하고 상태를 '해결됨/승인/반려' 등으로 변경하는 기능

# 11. 관리자 페이지 (신설)

## F-35 관리자 페이지

- **설명**: 관리자가 서비스 운영 전반을 관리하는 통합 페이지. 청년 가입 검수, 악성 유저 제재, 매칭 관리, 신고 처리, 증명서 발급 현황, 기기 상태를 한 화면에서 관리한다.
- **사용자**: 관리자
- **MVP 처리**: 관리자 페이지는 프론트엔드 화면만 구현하며, 더미 데이터를 기반으로 화면 흐름을 시연한다. 실제 API 연동은 후순위로 한다.

**구성 화면**

### F-35-1 관리자 대시보드

- **설명**: 승인 대기 청년 수, 신고 접수 건수, 진행 중 매칭 수 등 운영 현황을 요약해 보여주는 메인 화면
- **출력**: 승인 대기 청년 수, 미처리 신고/중단 요청 수, 활동 중 청년 수, 진행 중 매칭 수 요약 카드

### F-35-2 청년 가입 검수

- **설명**: `PENDING(승인 대기)` 상태의 청년 프로필을 검토하여 승인 또는 반려하는 화면 (F-33 연계)
- **사용자 시나리오**: 관리자가 승인 대기 목록에서 청년 프로필을 확인한 뒤 `APPROVED`로 변경하거나, 반려(`REJECTED`) 사유를 작성해 돌려보낸다.
- **입력**: 청년 ID, 검수 결과(승인/반려), 반려 사유
- **출력**: 검수 결과 반영 및 청년 상태 업데이트

### F-35-3 악성 유저 제재 관리

- **설명**: 신고가 누적되거나 운영 정책을 위반한 유저의 계정 상태를 `SUSPENDED`로 변경해 접근을 차단하는 화면 (F-33 연계)
- **입력**: 유저 ID, 제재 사유
- **출력**: 계정 상태 `SUSPENDED` 변경 및 해당 유저 로그인 차단

### F-35-4 매칭 관리

- **설명**: 생성된 매칭 목록과 상태(`MATCHED`/`IN_PROGRESS`/`ENDED`)를 조회하고 필요 시 조정하는 화면
- **출력**: 매칭 목록, 청년-어르신 정보, 매칭 상태

### F-35-5 신고 및 중단 요청 처리

- **설명**: 접수된 신고(F-31)와 매칭 중단 요청(F-32)을 확인하고 처리 상태를 변경하는 화면 (F-34 연계)
- **입력**: 신고/요청 ID, 처리 상태(해결됨/승인/반려), 관리자 메모
- **출력**: 처리 결과 반영 및 관련 사용자에게 알림

### F-35-6 증명서 발급 현황

- **설명**: 청년이 발급받은 사회참여 활동 증명서(F-30) 발급 내역과 발급 시리얼 번호를 조회하는 화면
- **출력**: 청년별 누적 활동 시간, 증명서 발급 내역, 발급 시리얼 번호(`DRDR-2026-XXXX`)

### F-35-7 기기 상태 관리

- **설명**: 어르신에게 제공되는 전용 태블릿의 배송 및 연동 상태(`READY`/`SHIPPING`/`DELIVERED`)를 조회하는 화면
- **출력**: 어르신별 기기 배송 상태 및 연동 상태

# 12. 회원가입 분기 흐름 요약

```
F-01 사용자 유형 선택
   │
   ├─ [청년으로 시작하기]
   │      → F-02 회원가입 (PENDING 상태로 생성)
   │      → F-06 청년 자기소개 프로필 등록 (F-07~10)
   │      → 관리자 승인 대기 → 승인 후 활동 시작
   │
   └─ [보호자로 시작하기]
          → F-02 회원가입
          → F-11 어르신 프로필 무상 등록
          → 어르신 매칭 가능 상태 등록 완료
```

# 13. 변경 이력 요약

| 항목 | 변경 내용 |
| --- | --- |
| 보호자 연락처 인증 | 삭제. 회원가입(F-02)의 휴대폰 번호 입력으로 본인 확인 갈음 |
| 어르신 프로필 등록 | '무상 서비스' 섹션에서 분리하여 청년 자기소개 다음(섹션 4)으로 이동. 회원가입 분기 흐름에 맞춤 |
| 어르신 프로필 구조 | 청년 프로필과 동일 구조 적용 (목소리 샘플 제외) |
| 관리자 페이지 | F-35 신설. 프론트엔드만 구현 |
| 유료 결제/구독 | 전면 삭제 (무상 서비스 원칙) |
| 현금 정산 | 삭제. 사회참여 활동 증명서(F-30)로 대체 |
| 기능 번호 | 보호자 인증 삭제로 F-12 이후 번호 한 칸씩 당김 (총 F-01~F-35) |
